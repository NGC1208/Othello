# CPU AI仕様

CPU側の石は黒、人間は白とする。

## 目的

- 初心者がルールを覚えながら対局でき、十分に勝つ機会がある強さを提供する。
- 完全なランダム着手ではなく、オセロらしい基本的な戦略を用いる。

## 思考ルーチンアルゴリズム

- CPU AIはα-β枝刈り付きミニマックス法を使用する。
- 規定の思考時間を超えないようタイムアウトを設ける。
- 探索開始前に合法手を1つ予備手として保持する。
- 探索は深さ1から3まで反復深化方式で行う。
- 0.5秒で探索を中断し、最後に完了した深さの最善手を返す。
- 深さ1の探索完了前にタイムアウトした場合は、予備手を返す。
- 「3手先」はCPU、人間、CPUの3着手を意味する。

## 評価関数

CPUから見た評価値を計算し、値が大きい局面を有利とする。

### 共通定義

- 合法手差 = CPUの合法手数 - 人間の合法手数
- 角差 = CPUが所有する角数 - 人間が所有する角数
- 危険マス差 = CPUが所有する危険マス数 - 人間が所有する危険マス数
- 石数差 = CPUの石数 - 人間の石数
- 角は盤面の四隅とする。
- 危険マスは、まだ取得されていない角に隣接する3マスとする。

### 序盤

空きマスが41以上の場合：

```text
評価値 =
  合法手差 × 20
  + 角差 × 100
  - 危険マス差 × 30
  + 石数差 × 1
```

### 中盤

空きマスが13～40の場合：

```text
評価値 =
  合法手差 × 15
  + 角差 × 120
  - 危険マス差 × 25
  + 石数差 × 3
```

### 終盤

空きマスが12以下の場合：

```text
評価値 =
  合法手差 × 5
  + 角差 × 120
  - 危険マス差 × 10
  + 石数差 × 20
```

### 終局時

- CPU勝利: `1,000,000 + 石数差`
- CPU敗北: `-1,000,000 + 石数差`
- 引き分け: `0`

### 同評価時

- 最も高い評価値の手が複数ある場合は、その中からランダムに選択する。
- 自動テストでは乱数シードを指定できるようにする。

## 性能上の制限

- 初期版の最大思考時間: 0.5秒
- 初期版の最大探索深度: 3手先
- 終盤完全読み: 行わない
- 対象端末: Windowsが動作する一般的なPC

## 受け入れ条件

- [ ] CPUは必ず合法手を選ぶ。
- [ ] 合法手がなければパスする。
- [ ] 同一条件で再現可能なテストを実行できる。
- [ ] 通常の局面で思考時間が0.5秒を大幅に超過しない。
- [ ] 終局局面では勝利・敗北・引き分けを最優先で評価する。
- [ ] 初期版では3手先まで探索する。
