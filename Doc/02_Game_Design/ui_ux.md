# UI / UX仕様

## 画面一覧

- 対局画面
- 難易度選択ダイアログ
- リザルト表示

## 初期版の対象外

- タイトル画面
- モード選択画面
- ポーズ画面
- 設定画面

ゲームを起動するとプレイヤー対CPUの対局画面を表示し、CPUが初手を指す前に難易度選択ダイアログを重ねて表示する。

## 難易度選択ダイアログ

- 起動後、対局を開始する前に必ず表示する。
- ダイアログ表示中は盤面を操作できず、CPUも思考・着手を開始しない。
- 「初級」「中級」「上級」の3つの選択肢を表示する。
- 初級: 目先で多くの石を取る、初心者らしいCPU。
- 中級: 2手先を読む、標準的なCPU。
- 上級: 3手先を読む、従来の強いCPU。
- 選択した難易度で直ちに対局を開始する。
- キャンセルして対局を開始する操作は用意しない。
- リザルトの「もう一度遊ぶ」を押した場合も難易度選択ダイアログを再表示する。

## 対局画面

- 盤面表示: 8×8マスの2D表示で完全俯瞰
- 現在の手番表示: 「CPUの手番」または「あなたの手番」を常時表示する。
- 難易度表示: 選択中の「初級」「中級」「上級」を常時表示する。
- 石数表示: 黒と白の石数を常時表示する。
- 合法手の表示: 表示しない。
- 最後の着手位置: 最後に石を置いたマスを枠で強調表示する。
- 対局中の戻る／やり直し: 初期版では実装しない。

## パス

- 人間に合法手がない場合、「あなたは置ける場所がないためパスします」と約1秒表示する。表示中は入力とCPU進行を停止し、表示後にCPUの手番へ自動進行する。
- CPUに合法手がない場合、「CPUは置ける場所がないためパスします」と約1秒表示する。表示中は盤面入力を停止し、表示後に人間の手番へ自動進行する。
- 両者に合法手がない場合はリザルト表示へ移行する。

## リザルト表示

- 勝敗を「あなたの勝ち」「あなたの負け」「引き分け」のいずれかで表示する。
- 黒と白の最終石数を表示する。
- 「もう一度遊ぶ」ボタンで難易度選択ダイアログへ戻り、選択後に初期盤面から再対局する。
- 「ゲームを終了」ボタンでアプリケーションを終了する。

## CPUエラー表示

- CPU思考が失敗した場合、まず現在盤面の合法手を予備手として使用し、対局継続を試みる。
- 予備手へ復旧できた場合はプレイヤーへエラーを表示せず、開発ログだけへ記録する。
- 合法手がなくパス条件を満たす場合は、通常のパスとして処理する。
- 合法手が存在するにもかかわらず予備手へ復旧できない場合は、対局を停止して「CPUの処理に失敗しました。もう一度対局を開始してください」と表示する。
- 復旧不能時の表示には「もう一度遊ぶ」と「ゲームを終了」を使用する。

## 入力方法

- マウス: 合法な空きマスを左クリックすると石を配置し、挟まれた相手の石が反転する。
- タッチ: 初期版では対応しない。
- ゲームパッド: 初期版では対応しない。
- キーボード: 初期版では対局操作に使用しない。

## 操作への反応

- 石を置いた演出: 石を即座に配置する。
- 石を裏返す演出: 成立したすべての方向の石を、同時に約0.2秒で反転させる。
- 不正な入力への反応: 合法手ではないマスをクリックしても何も起こさない。
- 勝敗演出: リザルト表示を画面上に重ねて表示する。追加の演出は初期版では実装しない。

## アクセシビリティ

- 色覚への配慮: 石は黒・白の明度差で識別し、最後の着手は黄色い枠で示す。
- 文字サイズ: 1280×720で読み取れる大きさとし、ウィンドウサイズに合わせて配置を調整する。
- アニメーション軽減: 初期版では切り替え設定を実装しない。反転演出は約0.2秒に限定する。

## ウィンドウ

- 基準解像度: 1280 × 720
- 起動方式: ウィンドウ表示
- ウィンドウサイズ変更: 許可する。
- 画面比率: 16:9を基準とし、サイズ変更時も盤面の縦横比を維持する。
- 全画面表示: 初期版では実装しない。
