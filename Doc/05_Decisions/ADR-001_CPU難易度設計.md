# 設計判断記録 001：CPU難易度を評価傾向と探索深度で分ける

- 状況: 採用
- 日付: 2026-06-18
- 決定者: プロジェクトオーナー

## 背景

従来の3手先探索CPUは、カジュアルプレイヤーに対して強すぎた。一方、合法手から無作為に選ぶ方式では、弱いというより意図のない不自然な着手になる。

特に従来の戦略評価は空いている角の隣接マスを強く避けるため、初心者が期待する「目先の石を多く取る」判断をほとんど行わなかった。

## 選択肢

1. 合法手から無作為に選ぶ。
2. 最善手から一定確率で準最善手や悪手を選ぶ。
3. 難易度ごとに評価傾向と探索深度を変える。

## 決定

難易度ごとに評価傾向と探索深度を変える。

- 初級: 1手先。反転数、石数、辺を重視し、角隣接マスへの減点を小さくする。
- 中級: 戦略評価を用いて2手先まで探索する。
- 上級: 戦略評価を用いて3手先まで探索する。
- 完全なランダム着手は行わない。
- 同評価の場合は座標順で決定し、同一条件で同じ手を選ぶ。
- 探索は難易度ごとの固定深度とし、実行時間による打ち切りは行わない。

## 影響

### 良い影響

- 初級が初心者らしい理由のある手を選ぶ。
- プレイヤーが自分に合う強さを選べる。
- 難易度差を探索深度と評価式として説明・テストできる。
- CPUの着手を再現でき、デバッグしやすい。

### 悪い影響・トレードオフ

- 初級専用の評価関数を保守する必要がある。
- 初級の人間らしさは重み調整に依存し、プレイテストによる改善が必要になる。
- 難易度選択UIと難易度別テストが追加で必要になる。
- 将来探索深度を増やす場合は、再現性を維持できる別の性能制御方法が必要になる。
