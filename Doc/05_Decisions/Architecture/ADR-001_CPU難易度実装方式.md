# ADR-001：CPU難易度を評価方式と固定探索深度で実装する

- 状況: 採用
- 日付: 2026-06-18
- 決定者: プロジェクトオーナー

## 関連DDR

- [[../Design/DDR-001_CPU難易度方針|DDR-001 CPU難易度方針]]

## 技術的背景

DDR-001で決定した3段階の難易度と初心者らしい初級CPUを、再現可能かつテスト可能な形で実装する必要がある。

時間制限付き探索では、端末負荷によって到達深度が変わり、同じ盤面でも異なる手を選ぶ可能性がある。また、完全なランダム着手や確率的な悪手選択では、判断理由が不明瞭になりデバッグも難しくなる。

## 技術要件・制約

- 初級・中級・上級の強さと判断傾向を区別できること。
- 同一盤面・同一難易度で同じ手を返すこと。
- Godot UIへ依存せず自動テストできること。
- 現在の最大探索深度3で実用的な応答時間に収まること。
- 合法手から無作為に選択しないこと。

## 選択肢

1. 全難易度で同じ評価関数を使い、確率で悪手を選ぶ。
2. 時間制限付き反復深化で、難易度ごとに思考時間を変える。
3. 難易度ごとに評価方式と固定探索深度を変える。

## 決定

難易度ごとに評価方式と固定探索深度を変える。

- 初級:
  - 初心者専用評価を使用する。
  - CPU自身の1着手後だけを評価する。
  - 反転数、石数差、角、辺を加点し、空いている角の隣接マスへの減点を小さくする。
- 中級:
  - 戦略評価を使用する。
  - CPU・人間の2着手を固定深度で探索する。
- 上級:
  - 戦略評価を使用する。
  - CPU・人間・CPUの3着手を固定深度で探索する。
- 中級・上級はα-β枝刈り付きミニマックス法を使用する。
- 同評価の場合は盤面左上から右下への座標順で最初の手を選ぶ。
- 実行時間による探索打ち切りは行わない。
- 評価式の重みと局面境界は名前付き定数で管理する。

## 影響

### 良い影響

- 難易度差を評価方式と探索深度として明確に説明できる。
- 同一条件で結果が再現でき、テストとデバッグが容易になる。
- 初級の判断傾向を戦略評価から独立して調整できる。
- DDRの「理由のある弱さ」を技術的に実現できる。

### 悪い影響・トレードオフ

- 初級専用評価と戦略評価の両方を保守する必要がある。
- 固定深度を将来増やす場合、応答時間の再評価が必要になる。
- 現在の盤面複製方式では、深い探索へ拡張すると性能改善が必要になる可能性がある。
